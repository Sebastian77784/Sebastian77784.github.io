4.3 Lenguaje máquina.

El lenguaje máquina es el conjunto de instrucciones binarias (formadas por ceros y unos) que puede ejecutar directamente un procesador. Es el nivel más bajo de programación y el único lenguaje que entiende el hardware sin necesidad de traducción adicional.

Características principales

  • Formado solo por bits (0 y 1)
  • Específico de la arquitectura (por ejemplo, una instrucción para procesadores Intel puede ser diferente a la de ARM)
  • Velocidad máxima de ejecución, ya que no requiere conversión
  • Muy difícil de programar manualmente
  • No legible para los humanos sin decodificadores o tablas de instrucciones

¿Por qué es importante en compiladores?

    En un compilador, la generación de código llega hasta un punto donde se convierte a lenguaje ensamblador, y finalmente este se transforma en lenguaje máquina. Por lo tanto, entender cómo se forma y ejecuta el lenguaje máquina es clave para:
  • Generar código eficiente
  • Optimizar instrucciones
  • Administrar recursos del sistema (memoria, registros, etc.)
Campo Descripción
Opcode Código de operación (por ejemplo, sumar, mover)
Operando Registro o dirección de memoria
Modo Modo de direccionamiento (directo, inmediato, etc.)

Cómo se genera el lenguaje máquina

  • Escribes en un lenguaje de alto nivel.
  • El compilador lo traduce a ensamblador.
  • El ensamblador convierte ese código en lenguaje máquina.
  • El sistema operativo carga el binario generado en memoria y lo ejecuta.
Ventajas Desventajas
Ejecución más rápida posible Muy difícil de entender y escribir por humanos
No necesita traducción adicional Altamente propenso a errores
Permite conocer exactamente qué hace el hardware Poco práctico para desarrollar software complejo
