1.7 Manejo de errores semánticos.

¿Qué es un error semántico?

Un error semántico ocurre cuando una sentencia del programa es sintácticamente correcta, pero carece de sentido lógico o viola las reglas del lenguaje desde el punto de vista del significado.

Ejemplo

Error semántico: no se puede asignar una cadena a una variable entera

Tipos comunes de errores semánticos

Tipo de error Descripción Ejemplo
Incompatibilidad de tipos Se realiza una operación entre tipos de datos incompatibles. int x = "texto";
Uso de variables no declaradas Se usa una variable que no ha sido previamente declarada. y = 5; sin declarar y
Redefinición de variables Se declara una variable que ya existe en el mismo ámbito. int a; int a;
Llamadas a funciones con parámetros incorrectos Número o tipo de parámetros no coincide con la declaración. f(1, "texto"); cuando f espera un solo entero
Modificación de constantes Intento de cambiar una constante o un valor definido como const. const int PI = 3.14; PI = 4;
Acceso indebido al ámbito Usar una variable fuera del ámbito donde fue declarada. Usar una variable local fuera de una función
División por cero En algunos lenguajes se detecta en tiempo de compilación si es evidente int a = 5 / 0;

¿Cómo se detectan los errores semánticos?

  • Consulta la tabla de símbolos para verificar si una variable ha sido declarada.
  • Verifica la compatibilidad de tipos para las operaciones y asignaciones.
  • Comprueba el ámbito de cada símbolo.
  • Valida el número y tipo de argumentos en llamadas a funciones.
  • Lanza errores o advertencias si detecta alguna violación de reglas semánticas.

Ejemplo de detección (pseudo-proceso)

Mensajes de error típicos

  • Error: variable 'y' no declarada en el ámbito actual.
  • Error: no se puede asignar un valor de tipo 'string' a una variable de tipo 'int'.
  • Error: número de argumentos inválido para la función 'calcular'.

¿Cómo se manejan estos errores?

  • El análisis semántico no solo los detecta, sino que intenta seguir adelante para encontrar más errores en una sola pasada.
  • Los compiladores suelen usar una estrategia de recuperación, como:
    • Omisión de la línea afectada.
    • Suposición por defecto (por ejemplo, asumir un tipo).
    • Inserción de tokens fantasma para seguir el análisis.

Importancia del manejo de errores semánticos

  • Mejora la calidad del código al detectar errores antes de la ejecución.
  • Ayuda al programador a entender el comportamiento incorrecto del código.
  • Permite al compilador continuar el análisis para mostrar todos los errores posibles, no solo el primero.
