4.2 Lenguaje ensamblador.

El lenguaje ensamblador (o assembly) es un lenguaje de bajo nivel que proporciona una representación simbólica de las instrucciones máquina que entiende directamente un procesador. Es específico para cada arquitectura (por ejemplo, x86, ARM, MIPS) y permite un control directo sobre el hardware.

Características principales

  • Traducción directa: Está muy cerca del lenguaje máquina.
  • Dependiente del hardware Cada instrucción en ensamblador equivale a una instrucción en código máquina.
  • Bajo nivel: Cada familia de procesadores tiene su propio conjunto de instrucciones y sintaxis.
  • Control absoluto del flujo y la memoria: Se puede manipular registros, direcciones, flags, etc.
  • No portátil: El código ensamblador para una arquitectura no funciona en otra sin modificaciones.

¿Por qué es importante en los compiladores?

  • Traducción final: Muchos compiladores generan código ensamblador como una etapa intermedia antes de convertirlo a binario.
  • Optimización: Permite estudiar cómo optimizar el código a nivel de instrucciones y recursos del procesador.
  • Depuración avanzada Algunos errores difíciles solo pueden analizarse correctamente revisando el ensamblador generado.

    • Elemento Descripción
    Instrucción Acción que debe realizar el procesador (mov, add, jmp, etc.)
    Operando Valor sobre el que actúa la instrucción (registro, memoria, constante)
    Registro Almacenamiento rápido interno al CPU (por ejemplo, eax, ebx)
    Etiqueta Marcador para instrucciones o direcciones (por ejemplo, inicio:)
