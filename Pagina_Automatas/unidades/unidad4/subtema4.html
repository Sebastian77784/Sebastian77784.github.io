4.3 Lenguaje máquina.

La administración de memoria es el proceso mediante el cual un programa (como un compilador o sistema operativo) controla, distribuye y reutiliza la memoria del sistema (RAM) para almacenar variables, instrucciones, estructuras y datos en tiempo de compilación o ejecución.

Objetivos principales

  • Asignar memoria a variables, constantes, estructuras, etc.
  • Liberar memoria cuando ya no se necesita (evitar fugas)
  • Maximizar el uso eficiente del espacio disponible
  • Evitar errores como el acceso a memoria no asignada o sobreescritura
Segmento Descripción
Stack (Pila) Memoria de funciones, variables locales, gestión LIFO
Heap (Montón) Memoria dinámica: malloc, new, free, delete
Código (Text) Donde se almacenan las instrucciones del programa
Datos (Data) Variables globales, estáticas e inicializadas

Procesos involucrados

Procesos Descripción
Asignación de memoria
  • En tiempo de compilación (variables estáticas)
  • En tiempo de ejecución (variables dinámicas, funciones)

    Liberación de memoria

  • Automática (recolector de basura en Java, Python)
    		•
    Gestión del espacio libre
  • Agrupar bloques pequeños libres para crear bloques más grandes
  • Evitar fragmentación (interna y externa)
    		•

    Métodos comunes de administración

    Método Descripción
    Estática Se decide durante la compilación (rápida, inflexible)
    Dinámica Se decide en tiempo de ejecución (flexible, más compleja)
    Por pila (Stack) Usada para funciones y llamadas recursivas
    Por montón (Heap) Usada para estructuras dinámicas (listas, árboles, etc.)

    Problemas comunes

    • Fugas de memoria (Memory leaks): memoria no liberada correctamente.
    • Fragmentación: pequeñas partes libres no aprovechadas eficientemente.
    • Overflows y underflows: se escribe fuera del espacio asignado (peligroso).
    • Acceso a punteros colgantes usar memoria que ya fue liberada.
