1.6 Generación de la tabla de símbolo y tabla de direcciones

¿Qué es la tabla de símbolos?

La tabla de símbolos es una estructura de datos que almacena información sobre los identificadores del programa fuente, como variables, funciones, constantes, tipos, etc.

    Es una especie de "diccionario" que el compilador usa para:
  • Verificar que los identificadores estén correctamente declarados.
  • Asociar atributos semánticos (tipo, dirección, alcance).
  • Facilitar la generación de código intermedio y final.

¿Qué contiene la tabla de símbolos?

Campo Descripción
Nombre Identificador (ej. x, suma)
Tipo Tipo de dato (ej. int, float, bool)
Clase Variable, función, constante, parámetro, etc.
Ámbito Local, global, dentro de función, etc.
Dirección Dirección relativa o absoluta en memoria
Tamaño Número de bytes ocupados (importante para arrays o estructuras)
Otros Valor constante, número de parámetros, tipo de retorno, etc.

¿Cuándo se genera la tabla de símbolos?

    Durante el análisis semántico, que viene después del análisis sintáctico. Cada vez que se encuentra una declaración de identificador, el compilador:
  • Extrae el nombre y tipo.
  • Asigna un espacio en memoria (dirección).
  • Inserta una entrada en la tabla de símbolos.

¿Qué es la tabla de direcciones?

    La tabla de direcciones (o asignación de direcciones) está relacionada con la tabla de símbolos y almacena:
  • La dirección de memoria (real o relativa) de cada identificador.
  • Las posiciones de memoria que ocuparán variables, constantes, parámetros, etc.
  • Se utiliza en la generación de código intermedio o código objeto.

Diferencia clave entre ambas

Tabla de Símbolos Tabla de Direcciones
Contiene información semántica Contiene ubicación en memoria
Útil para verificación de tipos, uso Útil para acceder a los datos en código
Más abstracta Más concreta para generación de código

Relación entre ambas

La tabla de símbolos incluye una columna de dirección → esa información puede usarse para construir la tabla de direcciones o usarse directamente desde allí en la generación de código.

Ejemplo

Supón el siguiente fragmento de código:

Tabla de símbolos

Nombre Tipo Clase Ámbito Dirección Tamaño
a int variable global 100 4
b float variable global 104 4
suma int funcion global - -
x int parámetro suma 200 4
y int parametro suma 204 4
z int variable suma 208 4
    Nota: Las direcciones son ejemplos. Puenden ser relativas o absolutas.
