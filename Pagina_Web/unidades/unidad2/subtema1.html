2.1 Notacioness

En el estudio de lenguajes formales, gramáticas y autómatas, las notaciones son formas estandarizadas de expresar lenguajes, gramáticas, expresiones regulares, producciones, etc. Estas notaciones permiten describir estructuras lingüísticas de manera precisa, clara y formal.

¿Para qué sirven las notaciones?

  • Para describir lenguajes formales de forma rigurosa.
  • Para construir analizadores léxicos y sintácticos.
  • Para validar estructuras gramaticales de lenguajes.
  • Para diseñar autómatas y compiladores.
  • Para probar propiedades de los lenguajes como regularidad, ambigüedad, etc.

¿Qué son las notaciones infija, prefija y postfija?

Son formas diferentes de escribir expresiones (por ejemplo, operaciones matemáticas), dependiendo de la posición del operador respecto a los operandos

2.1.1 Prefija

  • Forma: operador operando1 operando2
  • Ejemplo: + A B o * + 3 4 5

Explicación

En esta notación, el operador va antes de los operandos. Se evalúa de derecha a izquierda o usando recursión. Es muy utilizada en árboles sintácticos y algunos lenguajes funcionales

Ventajas Desventajas
No necesita paréntesis si se respetan las reglas. No es tan intuitiva para humanos.
Fácil de convertir a árbol de sintaxis abstracta (AST). Difícil de leer si no se está acostumbrado.

2.1.2 Infija

  • Forma: operando1 operador operando2
  • Ejemplo: A + B o (3 + 4) * 5

Explicación

Es la forma tradicional que usamos en matemáticas. Los operadores (como +, -, *, /) van entre los operandos. Para operaciones complejas, se utilizan paréntesis para indicar el orden correcto de evaluacion

Ventajas Desventajas
Es intuitiva y fácil de leer para los humanos. Ambigua sin paréntesis o reglas de precedencia.
Compatible directamente con la mayoría de los lenguajes de programación modernos (como C, Java, Python, etc.), ya que la sintaxis de estos lenguajes está basada en notación infija. Requiere mayor complejidad para ser procesada por computadoras.

2.1.3 Postfija

  • Forma: operando1 operando2 operador
  • Ejemplo: A B + o 3 4 + 5 *

Explicación

El operador va después de los operandos. Muy utilizada en máquinas de pila, evaluadores simples y algunas calculadoras programables (como las HP antiguas).

Ventajas Desventajas
No necesita paréntesis Poco intuitiva para los humanos, especialmente sin práctica previa.
Muy eficiente para evaluación con estructuras de pila. Difícil de escribir manualmente expresiones complejas, ya que no hay separadores visuales como paréntesis y se requiere conocer bien el orden.
